# 剧本转CSV工具 · 产品需求文档 v2.1

> 本版本为优化版,已修正 v1.0 中的安全、解析、边界处理问题,并根据 SayStation 平台真实模板校准了 CSV 输出格式,供 Claude Code 直接实现使用。

**v2.1 关键变更(相对 v2.0):**
- CSV 输出由 9 列改为 **8 列**(删除"视频提示词模式",因与"剧本分析模式"重复,平台会校验失败)
- CSV 头部以平台真实模板为准,固定为 6 行(带 BOM 的标题 + 5 行 `#` 注释)
- 明确标注了平台模板中存在的占位符 bug(`{{count}}`、`t('BatchCreateModal...')`),我们的输出**不要**包含这些

**目标平台:** SayStation (当前 UAT 环境:`uat.saystation.com`)

---

## 一、项目背景

公司短剧制作部门需将编剧产出的分集剧本,批量转换为符合 SayStation 平台上传格式的 CSV 文件。本工具让部门所有成员(无技术背景)通过一个网页界面完成全自动转换。

---

## 二、用户与使用场景

- **用户**:公司短剧制作部门全体成员
- **访问方式**:通过 GitHub Pages 链接在浏览器打开,无需安装任何软件
- **典型操作**:上传(或粘贴)12集剧本文档(含人物/场景链接表)→ 确认参数 → 下载 CSV → 上传 SayStation
- **浏览器支持**:Chrome / Edge / Safari / Firefox 最新版,桌面端

---

## 三、核心架构决策

### 3.1 不使用 AI,改为确定性解析

**决策理由:**
- 剧本格式高度结构化(`第X集:`、`X-Y 时间/场景 场景名`、`人物:A、B`、双空行分镜),用正则+字符串匹配可 100% 精确解析
- AI 方案存在三个致命问题:
  1. **安全**:纯前端无法隐藏 API Key,部署即泄露
  2. **成本**:每用户每次消耗 Token,且无法限流
  3. **稳定**:有幻觉风险,可能漏集或乱填
- 确定性方案:零成本、毫秒级、100% 可预测、GitHub Pages 即可部署

### 3.2 技术栈

| 层面 | 选型 | 理由 |
|---|---|---|
| 前端 | 原生 HTML + ES6 模块化 JS | 无构建步骤,易维护,加载快 |
| 样式 | Tailwind CSS (CDN) + 少量自定义 CSS | 快速实现专业界面 |
| docx 解析 | mammoth.js (本地化) | 浏览器端成熟方案 |
| 文件下载 | Blob + URL.createObjectURL | 原生 API |
| 参数持久化 | localStorage | 记住用户偏好 |
| 部署 | GitHub Pages + Actions | 零成本,自动部署 |

## 四、输入规格

### 4.1 剧本文件

**支持形式(互斥):**
- 文本框直接粘贴
- 上传 `.txt` 文件 (UTF-8 编码)
- 上传 `.docx` 文件

**互斥规则:** 若文件上传区已有文件,文本框置灰;文本框有内容时,上传按钮禁用。用户可点"清空"重置。

**内容结构(每集必须符合):**

```
第5集:清晨的越界
5-1 白天/内 林晨希卧室
人物:林晨希、苏夏妍
〔视觉状态〕……
▲[镜头描述] 剧情内容……
台词……

(双空行 = 分镜分割)

▲[镜头描述] 下一个分镜内容……
```

**格式定义(严格):**

| 元素 | 正则/规则 | 必需 |
|---|---|---|
| 集头 | 行首匹配 `^第\s*\d+\s*集[::]\s*(.+)$` | 是 |
| 分镜头信息行 | 行首匹配 `^\d+-\d+\s+\S+\s+\S+` | 是(每集至少一个) |
| 人物行 | 行首匹配 `^人物[::]\s*(.+)$`,名字用中文顿号`、`分隔(兼容半角逗号、中文逗号) | 是 |
| 分镜分隔 | 连续 2+ 个空行 | 集内分镜间必需 |
| 集间分隔 | 下一个集头出现即视为上一集结束 | 无硬分隔符 |

### 4.2 人物与场景链接表

**位置:** 整个剧本文档末尾,以明显分隔线或直接衔接最后一集。

**格式(每条一行):**

```
林晨希|https://...
苏夏妍|https://...
林晨希卧室|https://...
```

**分隔符兼容:** 竖线 `|` / 全角竖线 `|` 均接受。

**识别规则:** 
- 链接表起点:从文档末尾向前扫,连续遇到符合 `名称|URL` 格式的行即认定为链接表
- 链接表中**不应**出现中文句号或"第X集"字样,若出现则停止扫描
- 允许有空行,空行跳过

### 4.3 匹配逻辑

每集根据以下字段匹配链接:
1. **人物**:从每集人物行提取名字列表(去重)
2. **场景**:从每集所有分镜头信息行提取场景名(第三段),去重

匹配策略:
- 精确匹配优先
- 去除首尾空格
- 若链接表中缺失某人物/场景:记录警告,在 CSV 第三列该位置用 `[名称]|未找到链接` 占位,不中断处理
- 若链接表中有重复名称:取第一条,记录警告

## 五、输出规格

### 5.1 CSV 文件结构

共 **8 列**,一集一行:

| 列 | 字段名 | 内容 | 默认值 |
|----|--------|------|----------|
| 1 | 剧集 | `第X集-集名` 或 `第X集:集名`(与模板一致,建议用半角连字符 `-`) | 自动提取 |
| 2 | 剧情 | 全集所有分镜内容合并,原双空行保留作为分镜分隔 | 自动提取 |
| 3 | 资产库人物 | 本集人物+场景链接,每条间空一行 | 自动匹配 |
| 4 | 宽高比 | 用户选择 | `9:16` |
| 5 | 画风 | 用户选择 | `日漫二次元` |
| 6 | 模型 | 用户选择 | `HM-Jimeng SD2.0 Fast Official` |
| 7 | 分辨率 | 用户选择 | `720p` |
| 8 | 剧本分析模式 | 用户选择 | `剧本模式` |

> 说明:平台模板头部声称"包含 7 列",但实际接收第 8 列"剧本分析模式"(可选)。**不要输出第 9 列**"视频提示词模式",否则平台校验失败。

**第三列格式示例:**

```
林晨希|https://...

苏夏妍|https://...

林晨希卧室|https://...
```

### 5.2 CSV 头部(固定,照搬 SayStation 模板)

**必须输出的 6 行头部**(第 1 行带 UTF-8 BOM):

```
上传 CSV 文件批量创建项目
# 画面比例可选: 16:9 / 9:16
# 画风可选: 美式卡通 / 2D古风 / 3D古风 / 韩流一次元 / 现代都市 / 3D卡通 / 日漫二次元 / 中国工笔画 / 写实风格 / 彩色水墨
# 模型可选: say-video hot / HM-Jimeng SD2.0 Fast Official / say-video fast
# 分辨率可选: 720p
# 剧本分析模式可选(可留空): 批量创建项目 / 选择文件 / 下载模板文件
```

**紧接着输出数据行**(第 7 行开始是第 1 集,第 8 行是第 2 集,以此类推)。

**⚠️ 平台模板 bug 说明(不要照抄):**

平台下载的空白模板文件在第 7-10 行有占位符泄漏,**我们的输出必须不包含这些**:

- `请上传 CSV 格式文件，包含 7 列：...` ← 描述性文本,不是有效头部
- `校验通过 {{count}} 个项目` ← 未替换的模板变量
- `t('BatchCreateModal.batchCreate.hint4')` ← 未替换的 i18n key(实际应为剧情内容)
- `t('BatchCreateModal.batchCreate.hint5')` ← 未替换的 i18n key(实际应为人物链接)

这些是平台生成模板时的 bug,我们的工具要生成**干净的**文件。

### 5.3 CSV 技术细节(必做)

1. **编码:** UTF-8 with BOM (`\uFEFF` 开头),否则 Excel 打开显示乱码
2. **转义规则:**
   - 字段含 `,`、`"`、`\n`、`\r` 时,整个字段用双引号包裹
   - 字段内的 `"` 替换为 `""`
3. **换行符:** 统一使用 `\r\n`(Windows 兼容)
4. **文件名格式:** `批量导入_YYYYMMDD_HHmmss.csv`(精确到秒,避免同日覆盖)

### 5.4 输出约束

仅输出 `.csv`,不输出 `.xlsx`。

## 六、功能模块

### 6.1 文件上传模块

- 文本框(多行 textarea,高度 ≥ 300px)支持粘贴
- 文件上传支持 `.txt` / `.docx`
- 文件上传后自动读取内容显示到文本框(用户可二次编辑)
- `.docx` 通过 mammoth.js 提取纯文本,丢弃所有格式
- 读取失败(编码错误/损坏文件)给出明确错误提示

### 6.2 参数设置模块

用户可修改,**修改后保存到 localStorage,下次打开恢复上次配置**。提供"恢复默认"按钮。

| 参数 | 默认值 | 可选项 |
|------|--------|--------|
| 宽高比 | `9:16` | `16:9` / `9:16` |
| 画风 | `日漫二次元` | 美式卡通 / 2D古风 / 3D古风 / 韩流一次元 / 现代都市 / 3D卡通 / 日漫二次元 / 中国工笔画 / 写实风格 / 彩色水墨 |
| 模型 | `HM-Jimeng SD2.0 Fast Official` | say-video hot / HM-Jimeng SD2.0 Fast Official / say-video fast |
| 分辨率 | `720p` | `720p` |
| 剧本分析模式 | `剧本模式` | `剧本模式` |

> 所有下拉选项以常量形式集中维护在 `src/js/config.js`,方便后续 SayStation 新增选项时统一修改。
>
> **补充说明:** SayStation 平台有 "Global Config"(全局配置)功能,上传后可统一修改所有剧集的宽高比/画风/模型等参数。所以即使 CSV 中第 4-8 列填错,用户上传后在平台侧也能批量修正,我们的工具不必追求这些参数的 100% 准确。

### 6.3 解析模块(核心)

**流程:**

1. **预处理**
   - 统一换行符为 `\n`
   - 移除 BOM
   - 保留双空行结构(不 trim 全文)

2. **分离链接表**
   - 从文档末尾向前扫描
   - 连续匹配 `^(.+?)\s*[|\|]\s*(https?://\S+)\s*$` 的行加入链接表
   - 遇到不符合格式的非空行则停止,以此行为界,前半部分为剧本,后半部分为链接表

3. **切分集**
   - 按 `^第\s*(\d+)\s*集[::]\s*(.+)$` 正则扫描所有集头
   - 两个集头之间的内容归属前一集
   - 提取集号、集名

4. **解析每集**
   - 提取所有分镜头信息行(获取场景名)
   - 提取人物行,分割名字
   - 按连续 2+ 空行分割分镜
   - 合并所有分镜内容为单字符串(用双空行连接,保留原始分隔)

5. **匹配链接**
   - 遍历本集人物 + 场景(去重、保序)
   - 从链接表查对应 URL
   - 组装第三列字符串(条目间空一行)

6. **生成 CSV 行**
   - 按 8 列顺序填充
   - 对剧情和链接列做 CSV 转义

### 6.4 校验模块

每集处理前先校验,不符合则标记错误跳过:

| 错误码 | 含义 | 举例提示 |
|---|---|---|
| `E_NO_HEADER` | 集头后缺失分镜信息行 | "第5集找不到分镜头信息行(如 `5-1 白天/内 场景名`)" |
| `E_NO_CHARACTERS` | 缺人物行 | "第5集缺少人物行" |
| `E_NO_SHOTS` | 集内无任何分镜内容 | "第5集没有识别到分镜内容" |
| `E_BAD_EPISODE_NUM` | 集号重复 | "第5集出现多次,请检查" |
| `W_MISSING_LINK` | 链接缺失(警告,不跳过) | "第5集的人物'张三'在链接表中未找到" |

### 6.5 错误处理模块

- 错误分两级:**致命错误**(本集跳过)和 **警告**(继续但提示)
- 处理完成后在界面展示汇总:
  - ✅ 成功处理:第 1/2/3/4/6/7/8/9/10/11/12 集
  - ❌ 失败:第 5 集 - 缺少人物行
  - ⚠️ 警告:第 3 集 - 人物"王五"未找到链接
- 下载按钮在有至少一集成功时可用
- 全部失败时禁用下载,给出指引

### 6.6 下载模块

- 生成 CSV Blob,UTF-8 with BOM
- 通过 `<a>` 标签的 `download` 属性触发下载
- 文件名:`批量导入_YYYYMMDD_HHmmss.csv`

## 七、界面要求

### 7.1 布局(从上到下)

1. **Header 区**
   - 工具名称:`剧本 → CSV 转换器`
   - 副标题:`SayStation 批量上传格式`
   - 可选:简短使用说明展开按钮

2. **输入区**
   - 文件拖放/点击上传区 (`.txt` / `.docx`)
   - 或 文本粘贴区(textarea)
   - "清空"按钮

3. **参数设置区**
   - 6 个下拉选择器,横向紧凑排列(响应式 2-3 列布局)
   - "恢复默认值"按钮

4. **操作区**
   - 大号主操作按钮:`开始转换`
   - 处理中显示 loading 状态

5. **结果区**(处理后显示)
   - 汇总卡片:成功/失败/警告数量
   - 详细列表:每集状态
   - 下载按钮(粘性定位,始终可见)

### 7.2 视觉风格

- **整体调性:** 现代、专业、略带科技感,避免简陋
- **配色:** 浅色背景,主色调为深蓝或紫色渐变
- **字体:** 系统默认中文字体栈 (PingFang SC / Microsoft YaHei)
- **交互细节:**
  - 按钮 hover / active 状态
  - 文件拖入时上传区高亮
  - 处理中 loading 动画
  - 结果卡片淡入动画
  - 错误项红色、成功项绿色、警告项黄色

### 7.3 文案

- 全中文
- 错误提示使用非技术语言(不暴露正则、API 等术语)

## 十一、验收标准

1. 上传一份标准格式的 12 集剧本 → 下载得到 CSV,包含 6 行固定头部 + 12 行数据,共 8 列,内容正确
2. 上传一份第 5 集缺人物行的剧本 → 提示第 5 集出错,其他 11 集正常处理
3. 上传链接表缺某人物的剧本 → CSV 生成,第三列该位置显示"[张三]|未找到链接",并在界面警告
4. 修改参数后刷新页面 → 参数保持用户上次选择
5. 点击"恢复默认值" → 所有参数重置
6. 下载的 CSV 用 Excel 打开 → 中文不乱码,字段不错位
7. 空文档/只有链接表 → 友好提示,不崩溃
8. `.docx` 含复杂格式(表格/图片) → 仅提取文本,忽略其他
9. GitHub Pages 部署后打开正常工作,无跨域/加载错误
10. 生成的 CSV 头部 6 行与平台模板的 1-6 行完全一致(字符级比对),不包含模板文件里的 `{{count}}`、`t('...')` 等占位符
11. 实际上传到 SayStation 平台测试至少一次,确认能被识别并正确创建项目

## 十二、给 Claude Code 的实现指引

1. **先写核心解析器**(`src/js/parser/*`、`matcher/*`、`generator/*`),**不碰 UI**,用 Node.js 或浏览器控制台快速测试
2. **准备 3-5 个测试用例**(正常/缺人物/缺链接/集号重复/仅链接表),确保解析正确
3. **再写 UI**(`index.html` + `ui/*`、`io/*`),对接解析器
4. **最后处理部署**(GitHub Actions)
5. **开发顺序:** `config.js` → `utils/errors.js` → `parser/*` → `matcher/*` → `generator/*` → `io/*` → `ui/*` → `main.js` → `index.html` → `style.css`
6. **关键注意点:**
   - CSV 必须 UTF-8 BOM,否则 Excel 乱码
   - 双空行要用 `/\n\s*\n/` 而非 `/\n\n/`(兼容多空行)
   - 集头正则兼容中英文冒号、空格
   - 链接表分隔符兼容 `|` 和 `|`
   - 解析器纯函数,不依赖 DOM,方便测试
